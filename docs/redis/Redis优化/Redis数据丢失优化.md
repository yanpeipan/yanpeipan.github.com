数据一致性策略：最终一致，无论哪种持久化策略都无法保证数据完全不丢失

| 数据"丢失"场景 | 原因                                      |
| -------------- | ----------------------------------------- |
| CASE-1         | 业务逻辑导致的数据更新错误，形成脏数据    |
| CASE-2         | Redis服务器宕机，内存中的数据未完全持久化 |
| CASE-3         | 机房级别故障导致整机房数据完全丢失        |

## 优化策略
* 依赖DB进行兜底
* 依赖备份数据恢复
